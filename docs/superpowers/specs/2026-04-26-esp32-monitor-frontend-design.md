# ESP32 监控前端设计

## 概述

纯前端 HTML/CSS/JS 应用，浏览器直接访问 ESP32 提供的 MJPEG 视频流和控制接口。无需后端。

## 文件结构

```
esp32-monitor/
├── index.html     # 主页面
├── style.css      # 样式
└── app.js         # 逻辑
```

## 硬件配置（ESP32）

- IP: `10.0.0.110`
- 控制端口: `8080`
- 流端口: `8081`

## 功能

### 1. 摄像头开关
- `[开启]` 按钮 → `POST /control {"action":"on"}`
- `[关闭]` 按钮 → `POST /control {"action":"off"}`
- 开启时 MJPEG 流自动显示，关闭时视频停止或显示占位

### 2. 视频流
- `<img>` 标签绑定 `http://10.0.0.110:8081/stream`
- MJPEG multipart/x-mixed-replace

### 3. 舵机控制
- Slider 0-180° → `POST /control {"servo": 角度值}`
- 实时显示当前角度

### 4. 状态同步
- 每 5 秒轮询 `GET /control` 更新 UI

## API 交互

| 操作 | 方法 | 端点 | Body |
|------|------|------|------|
| 开启摄像头 | POST | `http://10.0.0.110:8080/control` | `{"action":"on"}` |
| 关闭摄像头 | POST | `http://10.0.0.110:8080/control` | `{"action":"off"}` |
| 控制舵机 | POST | `http://10.0.0.110:8080/control` | `{"servo": 45}` |
| 获取状态 | GET | `http://10.0.0.110:8080/control` | - |

## 响应格式

```json
{"status": "ok", "camera": "on"}
{"status": "ok", "camera": "off", "servo": 45}
{"status": "error", "reason": "init_failed"}
```

## 错误处理

- 连接失败：显示 "无法连接到 ESP32" 提示
- 请求超时：10 秒 timeout，弹窗提示
